# Plan Review — Step 1

## Verdict: REVISE

1. **The R001 checklist hydration is still not an implementable evidence contract.** `R001-2` says to bind conclusions to deterministic evidence, but it does not specify the route or stop conditions. The revised Step 1 plan must state that hard-session spacing is available only for athlete-identified sessions or activity/plan rows with sufficiently detailed, sourced intensity evidence; titles, aggregate load, date proximity, and age cannot classify a session as hard. It must explicitly add/use `compute_baseline` for one supported metric at a time and retain its status, sample counts, missing-day, freshness, method, and formula metadata—never combine those results into a score. For projections, use only copyable plan targets or athlete-supplied values, expose every `get_fitness_projection` assumption, and forbid treating its default `weekly_ramp_pct` / recovery-cadence values as plan evidence or a masters policy.

2. **Read-only is not yet made absolute.** The task expressly forbids write/delete calls, whereas the existing `plan_health_review` wording permits them after approval. Step 1 must require the new prompt and portable pack to say that `masters_plan_review` never calls write or delete tools, even after approval; any change remains a conditional, unapplied proposal. Add this exact boundary to the golden/boundary test so copying the older conditional-write language cannot regress the new prompt.

3. **The insufficient-evidence behavior and test plan remain too broad to verify.** Replace `R001-3`'s generic “every unavailable/ambiguous evidence trigger” with a test matrix: ambiguous/unavailable hard-session or plan detail; absent/invalid zones; short, partial, truncated, or missing history; missing/stale/partial wellness and missing/provider-native readiness; missing race context; and insufficient explicit load targets for a projection. For every case, require the affected dimension to have no comparison/conclusion, name the missing evidence, and ask one focused question. Preserve TP-235 semantics by labelling availability and requested duration as athlete-stated context rather than inferred constraints or session count.

4. **The Step 1 verification command is still a no-op in the ordered plan.** `go test ./internal/prompts -run 'MastersPlanReview'` selects no current test, while `masters_plan_review_test.go` is deferred to Step 2. Move the command to Step 2, or introduce and list a Step 1 test artifact before running it. The selected tests must cover the section/provenance contract, no-age/no-medical/no-score/absolute-read-only boundaries, hard-session fallback, baseline metadata, projection-default boundary, and each insufficient-evidence case above.
