# v0.3 Dogfood Prompt Set

This is the canonical write-path prompt set for TP-029 v0.3 dogfood validation. Prompts are written with redacted placeholders and must be run only against the dedicated test athlete account whose credentials come from `.env-dev`.

## Redaction conventions

- Replace athlete identifiers with `[TEST_ATHLETE]`; in committed notes use only the redacted prefix recorded in `STATUS.md`.
- Replace activity, event, workout, folder, plan, gear, sport-setting, and custom-item IDs with placeholders such as `[ACTIVITY_ID]`, `[EVENT_ID]`, `[WORKOUT_ID]`, `[FOLDER_ID]`, `[PLAN_ID]`, `[GEAR_ID]`, and `[CUSTOM_ITEM_ID]`.
- Replace real dates with shifted placeholders (`[YYYY-MM-DD]`) or relative windows that preserve ordering only.
- Replace private comments, workout names, thresholds, wellness measurements, and body metrics with synthetic values from these prompts.
- Record tool names, response byte counts, latency, `_meta` fields, and pass/fail judgments in findings; do not paste raw tool payloads, API keys, complete athlete IDs, or private account data.
- Each write prompt must round-trip through the corresponding read tool before being marked PASS. Keep raw transcripts ephemeral and delete them after extracting redacted evidence.

Run each prompt in a fresh or clearly segmented MCP client session with the local binary launched from the lane worktree and credentials exported from `.env-dev`. After every write, re-read with the named verification tool(s) and record only redacted evidence in `docs/dogfood/v0.3-findings.md`.
