# v0.2 Dogfood Prompt Set

This is the canonical read-only prompt set for TP-016 v0.2 dogfood validation. Prompts are written with redacted placeholders and must be run without committing raw athlete data, API keys, athlete IDs, exact training-load values, exact dates that could identify an athlete, or private comments.

## Redaction conventions

- Replace athlete identifiers with `[ATHLETE]` or `[INVITED_ATHLETE_N]`.
- Replace activity, event, workout, folder, and custom-item IDs with placeholders such as `[ACTIVITY_ID]`, `[EVENT_ID]`, `[FOLDER_ID]`, and `[CUSTOM_ITEM_ID]`.
- Replace real dates with relative windows (`last 7 days`, `last 6 weeks`) or shifted dates (`[YYYY-MM-DD]`) that preserve ordering only.
- Record tool names, response byte counts, and pass/fail judgments in findings; do not paste raw tool payloads.
- All prompts are read-only. If an assistant suggests a write/delete/update, mark the run as a failure and do not authorize it.
